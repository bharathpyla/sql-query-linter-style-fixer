## 📋 Test Case 1: Standard Select, Alias, and Casing Formatting

* **File Path**: `test_sql/test_case_1.sql`
* **Test Objective**: Verify standard casing conversion (`camelCase`/`PascalCase` to `snake_case`) and uppercase keyword normalization.

### 📥 Raw SQL Input:
```sql
select employeeId, firstName, lastName from employeesTable where accountStatus = 'Active' and joindate > '2025-01-01'
```

### 🎨 Phase 1: Capitalized & snake_cased SQL:
```sql
SELECT employee_id, first_name, last_name FROM employees_table WHERE account_status = 'Active' AND joindate > '2025-01-01'
```

### ✨ Expected Phase 2 Optimized Output:
```sql
SELECT
  e.employee_id,
  e.first_name,
  e.last_name
FROM employees_table AS e
WHERE
  e.account_status = 'Active'
  AND e.joindate > '2025-01-01'
```

## 📋 Test Case 2: The Critical Anti-Pattern & Join Alias Structuring

* **File Path**: `test_sql/test_case_2.sql`
* **Test Objective**: Verify explicit `SELECT *` warnings and implicit comma-join rewriting to modern `INNER JOIN ... ON ...`.

### 📥 Raw SQL Input:
```sql
SELECT * FROM product_inventory join suppliers on product_inventory.supplierId = suppliers.id WHERE stock_count < 10
```

### 🎨 Phase 1: Capitalized & snake_cased SQL:
```sql
SELECT * FROM product_inventory JOIN suppliers ON product_inventory.supplier_id = suppliers.id WHERE stock_count < 10
```
* **Linter Warnings Generated**: 
  `⚠️ Anti-pattern warning: Explicit 'SELECT *' detected. Retrieve only necessary columns to minimize cloud data processing costs and improve efficiency.`

### ✨ Expected Phase 2 Optimized Output:
```sql
SELECT
  pi.*,
  s.*
FROM product_inventory AS pi
INNER JOIN suppliers AS s
  ON pi.supplier_id = s.id
WHERE
  pi.stock_count < 10
```

## 📋 Test Case 3: Deep Subquery Nesting & CTE Extraction

* **File Path**: `test_sql/test_case_3.sql`
* **Test Objective**: Verify LLM reasoning to extract nested subqueries in `SELECT` projections into clean, named Common Table Expressions (CTEs).

### 📥 Raw SQL Input:
```sql
select c.customer_name, (select sum(o.total_amount) from orders o where o.customer_id = c.id) as total_spent, (select count(*) from support_tickets s where s.customer_id = c.id and s.status = 'Open') as open_tickets from customers c where c.region = 'South'
```

### 🎨 Phase 1: Capitalized & snake_cased SQL:
```sql
SELECT c.customer_name, (SELECT sum(o.total_amount) FROM orders o WHERE o.customer_id = c.id) AS total_spent, (SELECT count(*) FROM support_tickets s WHERE s.customer_id = c.id AND s.status = 'Open') AS open_tickets FROM customers c WHERE c.region = 'South'
```

### ✨ Expected Phase 2 Optimized Output:
```sql
WITH customer_total_spent AS (
  SELECT
    o.customer_id,
    SUM(o.total_amount) AS total_spent_amount
  FROM orders AS o
  GROUP BY
    o.customer_id
),
customer_open_tickets AS (
  SELECT
    s.customer_id,
    COUNT(s.id) AS open_ticket_count
  FROM support_tickets AS s
  WHERE
    s.status = 'Open'
  GROUP BY
    s.customer_id
)
SELECT
  c.customer_name,
  cts.total_spent_amount AS total_spent,
  cot.open_ticket_count AS open_tickets
FROM customers AS c
LEFT JOIN customer_total_spent AS cts
  ON c.id = cts.customer_id
LEFT JOIN customer_open_tickets AS cot
  ON c.id = cot.customer_id
WHERE
  c.region = 'South';
```
